# pcloud-components 组件文档生成提示词

## 目标
将组件的 index.zh-CN.md 源文档转换为精简的 skill 参考文档。

## 格式规范

```
# 组件名称

简短描述（1-2句话）

## 基础用法
// 完整可运行的 TSX 代码，40-80 行以内

## 场景二（可选）
// 另一个高频使用场景

## 场景三（可选）
// 第三个高频使用场景

## API

### PropsType
| 参数 | 说明 | 类型 | 默认值 |
|-----|-----|-----|-------|
// 完整列出所有属性

### SubType1
| 参数 | 说明 | 类型 |
|-----|-----|-----|
// 子类型表格

### SubType2
...

## 组件依赖
`组件A` · `组件B` · `组件C`
```

## 改写原则

### 保留
- 所有 props 完整列出（API 表格必须完整）
- 完整的 TypeScript 类型定义
- 核心 demo 示例（2-3 个）

### 精简/删除
- 去掉 front matter（`---` 块）
- 去掉组件特性列表（`- 📝 ···` 格式）
- 去掉"使用场景"段落
- 去掉"注意事项"段落
- 去掉"相关链接"（在线文档链接）
- 去掉冗余的文字描述，只保留必要的说明
- 组件依赖简化为单行：`A · B · C`

### Demo 选择标准
保留 2-3 个最常用、最高频的场景：
1. 基础用法（必须保留）
2. 其他高频场景（根据组件特性选择）

### 代码风格
- demo 代码使用完整的 TSX 格式
- import 语句明确写出
- 使用 `type` 关键字定义类型
- 避免过度简化的占位符代码

## 示例对比

**源文档（冗余）：**
```markdown
## 组件特性

- 📝 整合 DForm 表单，支持灵活的搜索配置
- 📊 基于 DTable 组件，提供高效的数据展示和分页
...

## 使用场景

1. **数据管理页面** - 快速实现数据的增删改查功能
2. **后台管理系统** - 简化各种资源的管理操作
...

## 注意事项

1. 如果自定义操作栏, 可以使用`ref`调用内部的方法...
```

**生成后（精简）：**
```markdown
## 基础用法
// 完整可运行代码

## API

### CRUDProps
| 参数 | 说明 | 类型 | 默认值 |
|-----|-----|-----|-------|
// 完整表格

## 组件依赖
`DForm` · `DTable` · `DModal`
```

## 执行步骤

1. 读取组件的 `index.zh-CN.md` 源文档
2. 提取标题和简短描述
3. 选择 2-3 个核心 demo
4. 完整列出 API 表格
5. 简化组件依赖为单行
6. 输出最终文档到 `references/docs/{ComponentName}.md`
